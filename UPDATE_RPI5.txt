UPDATE RPi5 - FIX Q-TABLE SENDING
==================================
Date: May 24, 2026
Commit: ae86f56

PROBLEM FOUND:
- Pico firmware expects Q-table for progressive learning (Rule-Based → FQL → DQN)
- wifi_bridge.py was set to "monitoring mode" and skipping Q-table send
- This caused Pico to connect but server never sent Q-table
- Connection kept resetting (TCP error -14)

FIX APPLIED:
- Restored Q-table sending in wifi_bridge.py
- Now sends: "QTABLE:[[...]]\\n"
- Waits for ACK: "ACK:QTABLE_LOADED\\n" or "ACK:QTABLE_ERROR\\n"
- 10 second timeout for ACK response

STEPS TO UPDATE RPi5:
=====================

1. Stop all services:
   cd /home/ubuntu/thesis
   sudo ./stop_all.sh

2. Pull latest code:
   git pull

3. Delete old Q-table to start fresh from Rule-Based:
   rm results/hasil_real/qtable.json
   rm results/hasil_real/dqn_buffer.json

4. Start all services:
   sudo ./start_all.sh

5. Monitor logs:
   # Terminal 1: Main log
   tail -f results/hasil_real/aquaculture_real.log

   # Terminal 2: Pico monitor
   tail -f results/hasil_real/pico_monitor.log

EXPECTED BEHAVIOR:
==================

Phase 1: Rule-Based (RB)
- Pico sends: DATA:ph_x1000,temp_x100,risk
- RPi5 logs data and trains FQL
- Pico shows: (RB) in serial output
- No Q-table sent yet

Phase 2: FQL Converged
- RPi5 sends: QTABLE:[[...9x4 floats...]]
- Pico receives and parses Q-table
- Pico sends: ACK:QTABLE_LOADED
- Pico switches to FQL mode
- Pico shows: (FQL) in serial output

Phase 3: DQN Ready (later)
- RPi5 sends updated Q-table from DQN
- Pico receives and updates
- Pico shows: (FQL) in serial output (still uses Q-table format)

TROUBLESHOOTING:
================

If Pico still shows connection errors:
1. Check RPi5 services are running:
   sudo systemctl status aquaculture

2. Check port 5000 is listening:
   sudo netstat -tlnp | grep 5000

3. Check WiFi hotspot is active:
   nmcli connection show --active | grep N3IWF_AQUA

4. Check Pico can reach RPi5:
   ping 10.42.0.206  # from RPi5

5. Check logs for errors:
   tail -50 results/hasil_real/aquaculture_real.log

If Q-table ACK timeout:
- Check Pico serial output for Q-table parsing errors
- Verify Q-table format is correct (9 rows, 4 columns)
- Check Pico firmware is latest version (commit 37eeadc)
